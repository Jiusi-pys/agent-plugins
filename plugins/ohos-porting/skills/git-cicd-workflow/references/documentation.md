# 文档记录规范

## Phase 报告

**位置**: `docs/progress/<track>/<stage>/<phase>_report.md`

**模板**: 见 `scripts/generate_phase_report.sh`

**内容**:
- Phase 信息（名称、层级、时间、commit）
- 改动概述
- 代码改动详情
- 编译和测试结果
- 遇到的问题和解决方案
- 参考的文档
- 下一步计划

## Stage 总结

**位置**: `docs/progress/<track>/<stage>/STAGE_SUMMARY.md`

**内容**:
- Stage 信息
- 包含的 Phases 列表
- 功能完成度
- 测试结果
- Review 意见
- Merge 信息

## Track 总结

**位置**: `docs/progress/<track>/TRACK_SUMMARY.md`

**内容**:
- Track 信息
- 包含的 Stages 列表
- 整体功能描述
- 关键成果
- Merge 信息
